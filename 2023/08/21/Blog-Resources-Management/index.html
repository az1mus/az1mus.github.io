管理博客资源

发表于 2023-08-21 分类于 网站、博客与Markdown

Hexo 很方便，如果能够定时将网站资源备份到自己的github页面上，就更方便了。

需要备份的资料

新版 Hexo 发布后，所有和网站相关的一手资源都在根目录下了，一目了然。
具有备份价值的文件或文件夹如下。

  1. _config.yml
    全局配置文件。必须备份。
  2. _config..yml
    主题配置文件，换主题之后要重新编辑。建议备份。
  3. source
    全部的博客内容。必须备份。

关于备份的建议

备份不能使用软链接或符号链接，因为那本质上不是有效的文件。
最简单的方法是每次手动复制然后推送，稍微高级一点的方法是写一个脚本，复制替换需要备份的文件。

之后，在自己 github 中新建一个私人仓库， clone 到本地后把软连接复制进去，提交修改。

相比于 Github 备份，更建议在诸如 Onedrive 的网盘上进行备份，使用mklink创建符号链接可以使同步更自动，更省心。

输入命令时，请使用绝对路径。
