BUU LFI COURSE 1

本地文件包含(LFI, Local File Include)，是 PHP 的常见写法。程序开发人员一般会把重复使用的函数写到单个文件中，需要使用某个函数时直接调用此文件，无需再次编写，这种文件调用的过程一般被称为文件包含[1]。

审计题目提供的 PHP 代码，发现不安全地使用了 include 函数。构造 payload：?file=<filename>便可以显示文件内容。

构造?file=/flag得到 flag。除此之外，还可以查看其他的文件，比如/var/log/nginx/access.log查看 nginx 文件记录。

除了 include 之外，PHP 还有其他常见危险函数。

题干的 PHP 代码指出，当类BUU析构时会输出 flag。那么，在程序中构造BUU即可。审阅题干，发现用到了unserialize($_POST['obj'])。为了让网页输出 flag，需要满足：

进入unserialize()所需的条件判断
serialize处理后的BUU类
BUU成员correct和input符合析构函数中需要的条件

为此，需要传 get 和 post 请求，然后求符合条件的md51 md52 obj在 post 里上传。

利用 Burp Suite 的 Repeater，可以编辑 post 请求并发送。获取md51 md52 obj，需要相关的 PHP 知识。

  • 数组的特殊性

    具体地，对于一个数组$arr，无论其中有什么元素，当其被双引号括起来解析后，他会被识别为字符串'Array'。在题目里，如果md51是数组，当它作为$_POST['md51']传入md5()函数后，实际上函数的返回值等于md5('Array')。这样，传入不同的数组md51和md52，就能满足条件md5($_POST['md51']) == md5($_POST['md52']) && $_POST['md51'] != $_POST['md52']。

  • 弱类型

    当 PHP 进行弱比较==时，系统会自动处理数据类型。在讲解PHP危险函数一文中，提到过符合科学计数法的字符串会被视为数字处理。也就是说，虽然很难构造两个完全一样的 md5，但是因为弱比较的特性，只需要构造两个哈希之后都是以0e开头的字符串即可，这是相对简单的。

    例如，字符串'QNKCDZO'和字符串's1836677006a'经过 md5 之后都是以'0e'开头。

  • 引用

    和 C++ 类似，PHP 中有引用的概念。当变量前加上&符号即可将其引用赋值给其他变量，之后两个变量同步变化，无论何时都相等。

    具体在题目中，声明BUU时，将其成员变量$input赋值为&$correct即可完成引用赋值，类似“绑定”，绕过析构函数中的值相等检测。
