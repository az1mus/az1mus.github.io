PHP 危险函数总结

发表于 2023-08-21 更新于 2023-08-22 分类于 CTF知识

搬运一下大佬对于PHP中危险函数的总结。

php 代码执行函数

eval

许多 webshell 利用 eval 实现。eval 把字符串当成代码执行。

官方文档在 eval 的页面中给出明确警告，指出“函数eval语言结构是非常危险的，因为它允许执行任意 PHP 代码。它这样用是很危险的。如果您仔细的确认过，除了使用此结构以外别无方法，请多加注意，不要允许传入任何由用户提供的、未经完整验证过的数据。”

所有传入的语句以分号结尾。作用域是调用 eval 时的作用域，所有修改均会被保留。

assert

在 PHP 8.0.0 之前，assert 倘若接收到一个字符串作为 assertion 参数，则会通过 eval 执行。这种行为在 PHP 8.0.0 被移除。

preg_replace

preg_replace 执行一个正则表达式的搜索和替换。

搜索subject中匹配pattern的部分，以replacement进行替换。

在 PHP 7 之前，pattern可携带 /e 修饰符，使 preg_replace 函数将pattern当成 PHP 代码。而subject通常由客户端提供，这样一来客户端可恶意构造subject参数来执行函数。

PHP 会解释双引号中的变量执行后的结果，因此"${function()}"是不安全的，其中的函数会被执行。因此如果replacement中希望使用 \k 来逆向替换，安全的做法是用单引号替代双引号。例如：

提交 ?h=[php]{${phpinfo()}}[/php] 代码会被执行，而如果其中的replacement为 'test('\\1')' 就不会执行了。

create function

创建动态参数并返回唯一名字。这个函数用到了eval并且性能很差，同时存在性能问题和安全性问题，已经在 PHP 8.0.0 被废除。官方文档建议使用原生匿名函数。

回调函数

调用$callback，其余参数为回调函数的参数。

如果$callback可自定义，则可以被利用，调用一些敏感函数如 system。

类似还有call_user_func_array和array_map。

包含函数

在 PHP 中，包含函数会直接运行文件。

  • require
  • include
  • require-once
  • include-once

include 和 require 的区别在于，如果 require 包含的文件不存在，PHP 会停止执行。包含分为本地包含(LFI, Local File Include)和远程包含(RFI, Remote File Include)，其中远程包含默认关闭，开启需要allow_url_include=on。被包含的文件会当做 PHP 代码执行，忽略后缀。也就是说，上传一个任意格式的文件，里面有一句代码，那么在包含时会被执行。

在 PHP 中，标点符号会干扰路径。

命令执行函数

exec, system, shell_exec, passthru

执行command参数指定的命令。command不能为空或包含 null 字节，必须是可执行的命令。当执行自定义二进制程序时，使用绝对路径且确保有访问权限。

显示输出结果，注意结果是处理过的。

相比于exec与system函数，显示原始输出。例如，调用pbmplus来输出图像流。

官方文档中给出明确警告，“当传入用户提供的数据到本函数时，应使用 escapeshellarg() 或escapeshellcmd() 来防止用户欺骗系统执行任意命令。”

escapeshellcmd

escapeshellcmd对字符串进行安全转义。但是在 Windows 下，在 PHP 5.6 之前，可能存在转义漏洞：将%1a转义成 bat 文件中的|。

反引号

在 SQL 中，加上反引号就不会有字段是关键字的问题。

在 shell 和 PHP 中，反引号本质上是操作系统执行命令，和exec类似。反引号在双引号中不起作用。

popen, proc_open

执行command指定的命令，产生一个单向进程管道。

类似popen但是控制程序执行的能力更强。如果只需要单向管道，使用popen即可。

注意command参数除了字符串还能是数组格式。进程将不通过 shell 直接打开。

文件执行函数

文件的读写可收到控制。例如，可以读取文件信息，写入 shell，删除 lock 文件从而安装覆盖等。常见的文件操作函数如下：

copy: 复制文件
file_get_contents: 将文件读入字符串
file_put_contents: 将字符串写入文件
file: 将文件读入数组中
fopen: 打开文件或 url
move_uploaded_file: 将上传的文件移动到新的位置
readfile: 输出文件
rename: 重命名一个文件或目录
rmdir: 删除目录
unlink & delete: 删除文件

特殊函数

通常作为注入对象或者注入内容。

phpinfo

输出 PHP 当前状态的大量信息，常用于在系统上检查配置设置和预定义变量。通常是调试工具。输出的信息包含 PHP 编译选项、启用的扩展、PHP 版本、服务器信息和环境变量（如果编译为一个模块的话）、PHP 环境变量、操作系统版本信息、path 变量、配置选项的本地值和主值、HTTP 头和PHP授权信息（License）。

symlink, readlink

对已存在的target建立名为link的符号链接。

返回符号链接的内容。如果path不是符号链接，函数将失效，在 Windows 上返回规范化的路径。

getenv, putenv

读取或写入环境变量，写入的环境变量仅存活于当前请求期间，请求结束后环境会恢复。

获取单个或全部环境变量。官方文档给出警告：“如果 PHP 在诸如 Fast CGI 之类的 SAPI 中运行，则此函数将始终返回由 SAPI 设置的环境变量的值，即使已使用 putenv() 来设置同名的本地环境变量。使用 local_only 参数返回本地设置的环境变量的值。”

assign为环境变量的设置，如foo=bar。

dl

载入指定extention_filename的 PHP 扩展。仅适用于 CLI 和嵌入式 SAPI，以及从命令行运行时的 CGI SAPI。extention_filename为扩展文件名，目录依赖于平台。

ini_get, ini_set, ini_alter, ini_restore

返回配置选项的值，注意是严格的字符串，如果在 ini 文件中有简化表示，则不能将返回值直接用于算术运算。

设置指定的配置选项的值，在脚本结束时恢复。ini_set能修改的选项是有限的，可选内容参考清单。ini_alter为ini_set的别名。ini_restore能够恢复配置选项到它的初始值。

is_numeric

检测指定的变量是否为数字或数字字符串。任意字母 e 周围的数字将会视作科学计数法。任何十六进制数将会被视作数字。

!is_numeric可以通过在数字前或后加%00空格字符、在数字后加%20空格字符或加任意非数字字符串来绕过。这么做之后，有关value的弱等于仍然会将value的数字部分用于正常比较。

将 SQL 注入代码转换成十六进制，能够绕过is_numeric，同时进行 SQL 注入。

intval

对value进行进制转换，返回value的 int 数值。不能用于 object。当base为0时，检测字符串value的前缀来决定是十六进制、八进制还是十进制。

当数值超出 int 上限后，返回 int 上限。可以利用这一点绕过回文验证。

in_array

在大海haystack中捞针needle。这里可以利用strict=false时的弱比较漏洞，在比较之前会进行自动类型转换，从而绕过in_array函数。

parse_str

如果设置了result参数，就把 parse 结果传入 result ，否则直接作用于当前作用域。在 PHP 8.0.0 之后，result参数是强制的。官方文档也给出明确警告：”极度不建议在没有 result 参数的情况下使用此函数， 并且在 PHP 7.2 中将 废弃 不设置参数的行为。PHP 8.0.0 起，result 参数是 强制 的。”

extract

将变量从数组导入到当前符号表中。检查是否作为合法变量名，检查是否与已有变量名冲突。官方文档给出明确警告：“不要对不可信的数据使用 extract() ，类似用户输入 （例如 $_GET、$_FILES）。如果这样做，要确保使用不会覆盖的 flags 值，例如 EXTR_SKIP ，并且要留意应该按照 variables_order 在 php.ini 里定义的顺序来提取。”

mb_parse_str, import_request_variables

据说能解析 GET/POST/COOKIE 并设置全局变量。目前只能解析 GET，检测其编码，转换为内部编码，令array包含转换过后的值。与parse_str类似，在 PHP 8.0.0 之后强制使用第二个参数。在之前的版本中，解析的值会覆盖到全局变量中。

弱化版的mb_parse_str，在 PHP 5 之后就不再受支持了。会覆盖全局变量。

glob

依照 libc glob() 函数的规则寻找所有与pattern匹配的文件路径，类似于一般 shell 所用的规则。不能作用于远程文件，且在某些旧系统上也无法工作。

攻击者可以使用 ../ 字符进行遍历攻击，也可以利用glob正则表达式，例如*.txt。

get_defined_vars, get_defined_constants, get_defined_funtions, get_include_files

可以利用这些函数实现无参数 RCE。更多无参数 RCE 可以看这篇文章和这篇文章。

ob_start

打开输出缓冲后，将不会输出内容，而是存储于内部缓冲区中。官方文档给出警告：“当有正在调用的回调函数时，一些 Web 服务器（例如 Apache）会改变脚本的工作目录。可以在回调函数中再把它改回来，例如 chdir(dirname($_SERVER['SCRIPT_FILENAME']))。”

可以将需要注入的命令作为回调函数，参数输出到缓冲区中，这样当执行ob_end_flush()时，或程序结束时，将会输出（执行）注入的命令。
