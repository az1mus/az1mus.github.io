PHP 危险函数总结

发表于 2023-08-21 分类于 CTF知识

搬运一下大佬对于PHP中危险函数的总结[1]。

危险函数

php 代码执行函数

eval()

许多 webshell 利用 eval() 实现。eval() 把字符串当成代码执行。

官方文档在 eval 的页面中给出明确警告[2]，指出“函数eval()语言结构是非常危险的，因为它允许执行任意 PHP 代码。它这样用是很危险的。如果您仔细的确认过，除了使用此结构以外别无方法，请多加注意，不要允许传入任何由用户提供的、未经完整验证过的数据。”

所有传入的语句以分号结尾。作用域是调用 eval() 时的作用域，所有修改均会被保留。

assert()

在 PHP 8.0.0 之前，assert() 倘若接收到一个字符串作为 assertion 参数，则会通过 eval() 执行。这种行为在 PHP 8.0.0 被移除。

preg_replace()

preg_replace() 执行一个正则表达式的搜索和替换。

搜索subject中匹配pattern的部分，以replacement进行替换。

在 PHP 3 之前（存疑），pattern可携带 /e 修饰符，使 preg_replace() 函数将pattern当成 PHP 代码。而subject通常由客户端提供，这样一来客户端可恶意构造subject参数来执行函数。

回调函数

调用$callback，其余参数为回调函数的参数。

如果$callback可自定义，则可以被利用，调用一些敏感函数如 system()。
