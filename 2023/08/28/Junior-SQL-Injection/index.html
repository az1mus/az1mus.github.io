粗略的 SQL 注入总结

发表于 2023-08-28 更新于 2023-09-04 分类于 CTF知识

常见的 SQL 注入可以帮助绕过用户名密码、构造 payload 获取表格元数据等，有助于攻击网站后端。

基础知识

一般的 SQL 注入点都会长这样：

倘若服务器对来自于用户的变量id没有加以限制，那么就可以通过构造id来进行一些规则之外的查询，例如查询其他表格中的敏感信息，或是 SQL 元数据。通常情况下，利用union select进行联合查询，注意如果union select之前查到就不会执行后面的了。

SQL 中的注释符号为#或--，需要通过注释来绕过基本的限制，最常见的做法是在构造的id末尾添加--+或--空格将剩余的语句注释掉。

SQL 通常分为字符型和数字型查找，字符型额外多一对单引号，需要手动添加上其中的一个（与原先的左半个匹配，原先的另一个被注释掉了）。

大小写

SQL 无视大小写，因此随机大小写，可以绕过一些简单的检测。

双写

某些过滤器会将字符串中的关键字替换为空，这时如果关键字外面套了一层，就会变成正常的一个关键字，例如SEselectLECT。

注释

使用--和#注释整行，使用/**/注释一部分。注意/*!*/在 MySQL 中能被执行，所以可以用来绕过过滤。

如果注释全被过滤了，使用闭合 SQL 语句代替注释符。

引号

除了上文提到的16进制替换，还能利用过滤器过滤引号的原理进行攻击。过滤器将引号'变为转义引号\'使其失去闭合的作用。通过在引号前加上%df将反斜杠和前面的%df组合形成 GBK 汉字；或者直接在'前加反斜杠把用来转义的反斜杠转义掉。根据不同过滤用 PHP 函数进行选择。

replace() 过滤 '\ ，将 ' 转化为 \' ，将 \ 转为 \\，将 " 转为 \" 。使用 GBK 处理。
addslaches() 返回在预定义字符之前添加反斜杠\的字符串。预定义字符：' , " , \ 。使用 GBK 处理。
mysql_real_escape_string()
